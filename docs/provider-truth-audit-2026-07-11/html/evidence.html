04 — Remaining risks

Where else optimism may hide

  • Plugins: provider capability is broader than Yoke's current management API.
  • Hooks: a native provider surface does not mean every hook form round-trips through Yoke.
  • Goals: Codex thread goals and Claude/Yoke loop goals have different ownership.
  • Models: requested model metadata is not effective-model attestation.
  • Workflows: Yoke's portable graph is useful but is not provider-native workflow execution.
  • CLI and unimplemented SDK profiles: metadata can describe a surface for which Yoke has no runnable adapter.

Every status row needs three proofs: provider supports it, Yoke lowers it, and a live run exercises that lowering.
