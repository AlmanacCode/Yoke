Provider truth audit · July 2026

Yoke is strongest when it translates mechanics, weakest when it claims semantic parity

Claude SDK subagents are a true provider mapping. Codex app-server delegation is real, but Yoke-declared custom agents are currently prompt-compiled rather than registered. The distinction must become impossible to miss.

01 — Direct answer

Your concern is valid

Yoke has three very different kinds of “works”: direct provider mappings, explicit provider-file generation, and model-followed prompt compilation. They currently sit too close together in the public story.
