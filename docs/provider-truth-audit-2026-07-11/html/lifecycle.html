Ephemeral compilation audit

Temporary provider bundles are correct for definitions—not for history

Yoke should compile one canonical agent package into a per-session runtime projection when a provider expects files. The bundle may disappear after the session closes. The provider transcript, Yoke run record, product job, and compilation recipe have different owners and lifetimes.

01 — Verdict

This is a standard adapter pattern, with one important correction

Claude explicitly accepts absolute local plugin paths. Codex app-server explicitly accepts non-persistent skill roots and process configuration overrides. Those are supported runtime injection seams, so generating isolated files is not filesystem trickery.

Architecture decisionCompile per session, not blindly per turn. A live session should use one immutable bundle. Concurrent sessions receive different roots. A resumed session rebuilds the same logical bundle from its durable recipe.
Codex exceptionSkills fit this design today. Named custom subagents do not yet have a proven app-server selection path. Generating their TOML is valid; claiming the child used it is not.

05 — Resume contract

A transcript is insufficient to recreate the harness

On resume, Yoke must recompile from durable source rather than retain a dead temporary path. Store a small deployment receipt beside the product run:

If the source changed, the caller chooses deliberately: resume with the pinned definition, resume with the latest and record drift, or start a fresh provider session. Yoke must not silently resume an old transcript under a different agent definition.

06 — Recommended product contract

Call it a runtime deployment, not merely a bundle

  1. Load and validate the canonical Yoke agent.
  2. Resolve provider surface and classify each requested feature.
  3. Compile one immutable, secret-free session deployment.
  4. Start the provider using direct APIs plus generated roots where necessary.
  5. Persist provider session ID, compiler version, source fingerprint, lowering receipt, result, and normalized events.
  6. Keep provider transcripts provider-owned; CodeAlmanac keeps its run ledger.
  7. On close, remove only the generated deployment.
  8. On resume, deterministically rebuild and compare fingerprints before continuing.

Ephemeral compilation is correct when it is reproducible, inspectable, session-scoped, secret-free, and independent from durable history.
